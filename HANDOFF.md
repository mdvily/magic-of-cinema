## 6. The rating system (the centerpiece)

Lives in `src/lib/rating.ts`. **MovieMan designed this himself.** Don't change it casually.

### Structure
- **5 subcategories**, each scored **0вҖ“20** popcorn buckets (whole numbers only):
  - рҹ“– Story
  - рҹҺЁ Visuals
  - рҹ‘Ҙ Characters
  - рҹҺӯ Acting
  - рҹҺө Music
- **Overall = sum** (0вҖ“100), **auto-computed at render time** (never stored)
- **No decimals**, no half-popcorn

### Verdict bands (MovieMan-authored labels)
| Score | Label | Emoji | Tone class |
|---|---|---|---|
| 90вҖ“100 | Cinematic Masterpiece | рҹҸҶ | `masterpiece` (amber-400) |
| 80вҖ“89 | Awesome | рҹ”Ҙ | `awesome` (orange-500) |
| 70вҖ“79 | Bueno | рҹ‘Қ | `bueno` (emerald-500) |
| 50вҖ“69 | Mid | рҹҳҗ | `mid` (slate-400) |
| 30вҖ“49 | No Bueno | рҹ‘Һ | `no-bueno` (rose-600) |
| 0вҖ“29 | Neck No! | рҹ’Җ | `neck-no` (stone-900) |

`verdictFor(overall)` returns the band, `verdictClasses(tone)` returns Tailwind classes for the score badge.

### Validation
- Astro side: zod constrains 0вҖ“20 ints in `src/content.config.ts`
- Tina side: `validateBucket()` in `tina/config.ts` rejects non-integers and out-of-range

### Custom field: MoviePicker
Lives at `tina/components/MoviePicker.tsx`. React component, ~280 lines.

**What it does:** Renders a TMDb search input. As MovieMan types, it queries `/api/tmdb-search`, shows a dropdown of results with posters/year/overview. On click, it fetches `/api/tmdb-details` and autofills sibling form fields.

**Three "shapes"** (set via `field.options.shape`):
- `"movieObject"` вҶ’ fills movie.title, year, director, runtime, posterUrl, backdropUrl, tmdbId, genres
- `"tierCard"` вҶ’ fills card.title, posterUrl
- `"top10Item"` вҶ’ fills item.title, year, posterUrl

**Critical implementation detail:** The dropdown is rendered via **React portal into `document.body`** with `position: fixed` and `z-index: 2147483647`. This is because Tina's form layout has CSS containment that clips/blends absolutely-positioned children. We tried `position: absolute` (clipped) and `position: fixed` without portal (still blended). Portal + max z-index was the only thing that worked. **Don't refactor this back without solving the same problem.**

### Known Tina quirks (operational)
- When you change the schema in `tina/config.ts`, you must regenerate `tina/tina-lock.json` (run `npm run dev` briefly to regenerate, then commit).
- Tina Cloud sometimes lags re-indexing after schema changes. Symptom: Vercel build fails with `"local GraphQL schema doesn't match the remote"`. Fix: open the project's [configuration page](https://app.tina.io/projects/d3b76154-aaea-48b8-b45d-c74e09dc3d1a/configuration) and click **"Refresh Branches"**, then redeploy.
- Tina commits to GitHub with a generic message `"TinaCMS content update"` вҖ” these will dominate your git history. Filter with `git log --invert-grep --grep="TinaCMS content update"` to see real changes.
- After every Tina save, `tina-lock.json` may be rewritten with no actual diff вҖ” this is fine, ignore.

## 8. API endpoints (serverless)

Three Astro routes use `export const prerender = false` and run as Vercel serverless functions.

### 8.1 `GET /api/tmdb-search?q=<title>`
- Source: `src/pages/api/tmdb-search.ts`
- Proxies TMDb search. Returns slim `{ results: [{ id, title, year, posterUrl, overview }] }` (top 10).
- Cache: `s-maxage=300`
- Auth: none (TMDb key stays server-side)

### 8.2 `GET /api/tmdb-details?id=<tmdbId>`
- Source: `src/pages/api/tmdb-details.ts`
- Fetches full movie details + credits, extracts director from crew.
- Returns `{ tmdbId, title, year, director, runtime, posterUrl, backdropUrl, genres }`
- Cache: `s-maxage=3600`

### 8.3 `POST /api/notify-deploy?secret=<NOTIFY_SECRET>`
- Source: `src/pages/api/notify-deploy.ts`
- **Purpose:** Receives Vercel "deployment succeeded" webhook, forwards a Telegram message to Mike.
- **Auth:** Query-param shared secret (`NOTIFY_SECRET`).
- **Only fires on:** `type === "deployment.succeeded"` AND `target === "production"`.
- **Message includes:** Deploy URL, commit message, author, canonical site URL.
- **Status:** Endpoint is live and tested. **Vercel-side webhook configuration was never completed** вҖ” see В§10.
- Also supports `GET` (returns a plain-text "endpoint live" message; useful for verifying it's reachable).

## 14. Known issues and quirks

### Visual / content
1. **Homepage shows empty sections** for Tier Lists, Top 10s вҖ” those collections have zero entries. The homepage code (`src/pages/index.astro`) checks `if (s.summary)` before rendering, so empty types just don't show вҖ” currently rendering only "Latest Review", "Latest Hot Take", "Latest Trivia" sections. **Working as designed**, but it makes the page sparse.
2. **MovieMan uses `#` everywhere** in his review body markdown вҖ” see В§13. Should be `##` for sections. Visually loud but not broken.
3. **No featured image / hero on review pages** вҖ” just a poster on the side. Could be richer.
4. **No tag pages** вҖ” `tags` are stored on posts but there are no `/tags/<tag>` routes for browsing by tag.
5. **No search** вҖ” site has no client-side or full-text search.

### Operational
6. **Giscus GitHub app not installed** (see В§10) вҖ” comments component renders but visitors can't post yet.
7. **Vercel deploy webhook for Telegram not wired** (see В§10) вҖ” endpoint is live but nothing's calling it.
8. **Tina body editor may not preserve all hand-written markdown** on save. Specifically, the body field uses rich-text mode (`isBody: true`) which serializes back to markdown. Most cases are fine but exotic markdown (custom containers, footnotes) may be lossy. If MovieMan complains about formatting changing after he saves, this is the cause вҖ” consider switching `format: "md"` to `format: "mdx"` in `tina/config.ts`.
9. **Tina commits** dominate git history (every save = one "TinaCMS content update" commit). Real changes are buried. Use `git log --invert-grep --grep="TinaCMS content update"`.
10. **Vercel uses Node 24**, local development on Mike's machine is Node 25. Astro warns but it's harmless.

### Architectural / future
11. **`/admin` is publicly reachable** but requires Tina Cloud login to do anything. Acceptable.
12. **No analytics.** No tracking pixels. If Mike wants page-view counts later, the privacy policy says we'd document it.
13. **`PROJECT.md`** in the root is an older build log. Treat as historical; this `HANDOFF.md` is the canonical doc going forward.

## 16. How to make a code change (for future-you)

1. `cd ~/Documents/.../projects/magic-of-cinema` (or wherever you have it locally)
2. `git pull` to sync (Tina commits constantly)
3. `npm install` if you haven't (or after a dependency change)
4. **For local dev with CMS:** `npm run dev` вҶ’ site at `http://localhost:4321`, Tina at `http://localhost:4001`
5. **For local dev without CMS:** `npm run dev:astro` (faster, no Tina overhead)
6. **For local build sanity check:** `npm run build` (runs `tinacms build && astro build`; needs Tina env vars)
7. Commit. Push to `main`. Vercel deploys.
8. Watch deploy: `vercel ls` or https://vercel.com/mike-ds-projects-0d75113a/magic-of-cinema

### If you change the Tina schema:
1. Edit `tina/config.ts`
2. Run `npm run dev` briefly to regenerate `tina/tina-lock.json`
3. Commit BOTH `tina/config.ts` AND `tina/tina-lock.json`
4. Push
5. If Vercel build fails with "schema doesn't match remote," open Tina Cloud config page and refresh branches, then redeploy

### If you change the Astro content schema (`src/content.config.ts`):
- Existing markdown files must still validate against the new schema, OR you migrate them
- Test locally with `npm run build` before pushing вҖ” Astro will fail the build if any post violates the schema

### Telegram check after big changes:
After a successful deploy, if the webhook is wired, you should get a Telegram message from `@MikeD_rocky_bot` to Mike's chat (ID `8631789252`).
